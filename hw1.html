Question1:

For the three categories of web software technology we've just defined, you are asked to find out which category so-called "Chrome plug-in" technology belongs to. Please describe the basic idea of the technology, and give your reasoning on its position in the technology categories.

Answer to question 1:

I think "Chrome plug-in" belongs to Distributed Object/Component Technology. The reason is:

"Chrome plug-in" refers to software intended to add new features/function to Chrome borwser. Similar to CORBA, users can install plug-ins whenever they like. Meanwhile, plug-ins can call functions provided by the browser. They can cooperate with each other, even though they are developed with different languages, OS and tool sets, but they can be independent, too.

In all, "Chrome plug-in" fits the requirement on time, space, platform and awareness mentioned in the "Distributed Object Technology" section. So they are one of them.

Question3(optional):

Open Question: of those client side technologies, why Java Applet is getting dead and Javascript success? Try to explain from the "REST" perspective.

Answer to question3

My reasons are as follows:

  1. In REST style, the client end and the server end are separated. So the need of software intended for the client end, which is usually a browser, is huge. Java Applet is based on Java, and it is harder to learn Java than to learn javascript because the former is much more complicated.
  2. Java Applet is self-contained and only react to commands within its own boundaries. In other words, its capability to interact with the browser is not as good as javascript. It performs well in terms of creating games, 3D demonstrations and math tools, but it is not the case when it comes to handle http requests. Whereas, in REST style, we rely on http requests to handle the resources. Javascript, undoubtedly, is a better choice.
  3. Java Applet must be compiled into .class files and runs in JVM, so one applet is an indivisible whole. This feature results in a lack of flexibility. However, you can run only one or two lines of javascript code in the browser console, just changing the background color of one single page. Therefore, javascript can better realize the Code-on-Command Constrain.
